A tiny Bun-powered CLI that opens any .md file as a local, drag-and-drop kanban board. Edit in the browser, edit in your editor — both stay in sync, every change writes back to the file.

Built for plain-text people.

No database. No sync server. Just a CLI, a markdown file, and your editor of choice.

Markdown is the source of truth

Headings become columns, checkbox lines become cards. Diff-friendly canonical output keeps git PRs reviewable.

Live two-way sync

Drag in the browser, the file updates in milliseconds. Edit the file in vim, the board reloads.

AI-friendly

One-click "copy prompt" hands an agent the file path plus your custom instructions. Watch it tick items off live.

Themes & icons

16 palette colors that tint the page; 41 column icons with search. Boards stay readable, never gaudy.

Per-card & per-column deep links

Copy a link to any card, column, or the file on disk. Navigating to the link scrolls and flashes the target.

Tiny and fast

~1.7k lines of TypeScript. One Bun-built binary. No Electron. No cloud. localhost only.

Quick start.

One command, no install. If board.md doesn't exist yet, kanban-cli creates a starter board with a default instructions block at the top.

Read the install guide 
# run instantly
$ npx kanban-cli@latest board.md
▸ kanban-cli is running
▸ File:  /Users/you/board.md
▸ Board: http://127.0.0.1:4177

# or with bun
$ bunx kanban-cli@latest board.md

# or install globally as `kb`
$ npm i -g kanban-cli
$ kb board.md

The shape of a board.

Headings turn into columns. Checkbox lines turn into cards. The bit before the first heading becomes a global instruction block — handy for handing off to AI assistants.
